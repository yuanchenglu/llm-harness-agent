## 摘要

自主智能体长期以来一直是学术界和工业界的研究焦点。此前的研究往往侧重于在孤立环境中训练知识有限的智能体，这与人类的学习过程存在显著差异，使智能体难以实现类人决策。近期，通过获取海量的网络知识，大语言模型（LLM）在实现人类水平智能方面展现了潜力，引发了基于 LLM 的自主智能体研究热潮。在本文中，我们对这些研究进行全面综述，从整体视角对 LLM 自主智能体进行系统性梳理。我们首先讨论 LLM 自主智能体的构建，提出了一个涵盖大多数先前工作的统一框架。然后，我们概述了 LLM 自主智能体在社会科学、自然科学和工程领域的多样化应用。最后，我们深入探讨了 LLM 自主智能体常用的评估策略。基于先前的研究，我们还提出了该领域的几个挑战和未来方向。

**关键词：** 自主智能体，大语言模型，人类水平智能

## 1. 引言

"自主智能体是处于环境中并构成环境的一部分的系统，它感知环境，随时间推移采取行动，以追求自身目标，从而影响其未来感知到的东西。" —Franklin and Graesser (1997)

自主智能体长期以来被认为是实现通用人工智能（AGI）的有前途的方法，期望其通过自我导向的规划和行动完成任务。在先前的研究中，智能体被假定为基于简单和启发式的策略函数行动，并在孤立和受限的环境中进行学习 [1–6]。这些假设与人类的学习过程存在显著差异，因为人类的心智高度复杂，个体可以从更广泛的环境中学习。由于这些差距，先前研究获得的智能体通常远未达到复制人类水平的决策过程，特别是在不受约束的开放域场景中。

近年来，大语言模型（LLM）取得了显著成功，展现了实现类人智能的重大潜力 [5–10]。这种能力来自于利用全面的训练数据集和大量的模型参数。基于这一能力，越来越多的研究将 LLM 用作中央控制器来构建自主智能体，以获得类人的决策能力 [11–17]。与强化学习相比，LLM 智能体拥有更全面的内部世界知识，使其即使在没有特定领域数据训练的情况下也能执行有根据的行动。此外，LLM 智能体可以提供自然语言接口进行人机交互，提供更大的灵活性和增强的可解释性。

沿着这一方向，研究人员开发了许多有前景的模型（见图 1 概述），其关键思想是赋予 LLM 以记忆和规划等人类能力，使其行为类似人类并有效完成各种任务。此前，这些模型是独立提出的，很少有努力对它们进行整体总结和比较。然而，我们相信，对这一快速发展领域的系统性总结对于全面理解它具有重要意义，并有助于启发未来的研究。

在本文中，我们对 LLM 自主智能体领域进行了全面综述。我们围绕三个关键方面组织本综述：LLM 自主智能体的**构建、应用和评估**。对于智能体构建，我们关注两个问题：(1) 如何设计智能体架构以更好地利用 LLM；(2) 如何激发和增强智能体的能力以完成不同的任务。直观地说，第一个问题旨在为智能体构建"硬件基础"，而第二个问题则专注于为智能体提供"软件资源"。对于第一个问题，我们提出了一个统一的智能体框架，可以涵盖大多数先前的研究。对于第二个问题，我们总结了智能体能力获取的常用策略。除了讨论智能体构建外，我们还系统性地概述了 LLM 自主智能体在社会科学、自然科学和工程领域的应用。最后，我们深入探讨了评估 LLM 自主智能体的策略，重点关注主观和客观策略。

总之，本综述对 LLM 自主智能体这一新兴领域的现有研究进行了系统性回顾，并建立了全面的分类法。我们的焦点涵盖三个主要领域：智能体的构建、应用和评估方法。借鉴大量的先前研究，我们识别了该领域的各种挑战，并讨论了潜在的未来方向。我们期望本综述能够为 LLM 自主智能体的新进入者提供全面的背景知识，并鼓励进一步的突破性研究。

## 2. LLM 自主智能体的构建

LLM 自主智能体被期望通过利用 LLM 的类人能力来有效执行多样化的任务。为实现这一目标，有两个重要方面：(1) 应设计什么样的架构以更好地利用 LLM；(2) 在给定架构设计的情况下，如何使智能体获得完成特定任务的能力。在架构设计的背景下，我们对现有研究进行了系统性综合，形成一个全面的统一框架。至于第二个方面，我们根据是否对 LLM 进行微调来总结智能体能力获取的策略。将 LLM 自主智能体与传统机器学习进行比较，架构设计类似于定义网络结构，而能力获取类似于学习网络参数的过程。在以下各节中，我们将更详细地探讨这两个方面。

### 2.1 智能体架构设计

LLM 的最新进展已经证明了它们在以问答（QA）形式完成各种任务方面的巨大潜力。然而，构建自主智能体远不止于 QA，因为它们需要扮演特定角色，并自主感知和从环境中学习，像人类一样自我进化。为了弥合传统 LLM 与自主智能体之间的差距，一个关键方面是设计合理的智能体架构。

以前的工作提出，智能体必须具备三个核心组件：**记忆**、**规划**和**行动**。受此启发，我们提出了一个统一的框架，该框架还额外强调了 LLM（大脑模块）的关键作用。在我们的统一框架中，LLM 自主智能体由四个核心模块组成：

#### 2.1.1 大脑（Brain）

大脑模块主要由 LLM 组成，是智能体的核心推理和决策引擎。LLM 的几个关键特性使其适合作为智能体的大脑：

**自然语言理解。** LLM 经过大规模文本语料的训练，展现出强大的自然语言理解和生成能力。这使得智能体能够理解人类指令、解析复杂查询、并以自然语言回应——这是与传统智能体相比的关键优势。

**知识。** 经过大量数据预训练的 LLM，存储了跨越多个领域的广泛世界知识，包括常识、科学事实、历史事件、文化参考等。这种广泛的知识基础使智能体能够在没有特定领域微调的情况下做出有根据的决策。

**推理。** 先进 LLM 展现了令人印象深刻的推理能力——它们可以进行逻辑推理、数学推理和常识推理。通过思维链（CoT）提示 [18, 19]，LLM 可以逐步分解复杂问题并系统地推导出解决方案。思维树（ToT）[20] 和思维图（GoT）[21] 等更高级的推理技术使 LLM 能够探索多条推理路径并评估哪些路径最有可能产生正确的解决方案。

**规划。** 作为大脑模块的一部分，LLM 可用于将复杂任务分解为更小、更易管理的子任务，并为完成这些子任务创建序列计划。这在 LLM 被用作智能体控制器的场景中尤为有用。

**泛化。** 与传统 AI 系统不同——后者通常需要针对每个特定任务进行大量重新训练——LLM 可以泛化到新任务，只需通过上下文学习在提示中提供少量示例。这种灵活性使 LLM 智能体比传统智能体更具多功能性。

**角色扮演。** LLM 可以在上下文中扮演特定角色，使其行为与所需输出的特征一致。此能力允许创建专门的智能体——例如：通过提示 "You are a helpful teacher" 作为教师，或 "You are a strict critic" 作为评论者。

#### 2.1.2 记忆（Memory）

记忆模块在 LLM 智能体中起着关键作用。它将来自环境的观测和 LLM 自身的思维过程存储为内部日志，并在决策过程中检索相关的过往经验，以指导智能体未来的行动。根据记忆持续的时间长度和功能，记忆可以归类为两种主要类型：

**短期记忆。** 短期记忆存储最近的感知和行动。它包括当前任务中来自环境的观测以及在当前决策过程中产生的 LLM 思维——通常以注意力机制的键值对形式存储在 LLM 的上下文窗口中。然而，上下文窗口的长度是有限的，因此将短期记忆保持在 LLM 的上下文大小内需要高效的存储和检索技术。

**长期记忆。** 长期记忆存储智能体在整个生命周期中积累的重要经验。它通常使用外部向量存储来实现，其中经验被嵌入为向量，并通过相似度搜索进行检索。长期记忆使智能体能够回忆远距离的知识和过去任务中的教训。

**记忆操作。** 与记忆相关的主要操作包括：
- **记忆存储**：将信息写入记忆，包括来自环境的观测和 LLM 生成的思考
- **记忆检索**：基于相关性、新近性和重要性等因素从记忆中读取和提取相关信息
- **记忆反思**：综合和抽象过去的记忆，以获得更高层次的洞察和教训

#### 2.1.3 规划（Planning）

规划是人类解决问题的基本特征。它为智能体提供结构化的方法来处理复杂任务。我们根据在决策过程中是否涉及反馈，将规划方法分为两类：

**无反馈规划。** 智能体基于初始计划执行动作，不在执行过程中修改计划。这种方法简单但在环境变化时可能不够灵活。常见的策略包括：
- 将任务分解为子任务，并为每个子任务生成具体的行动计划
- 使用 LLM 为先前确定的子任务生成完整的步骤序列

**有反馈规划。** 智能体在执行过程中接收反馈，并相应调整其计划。这使智能体能够在出现不确定性或错误时进行修正。常见的策略包括：
- **ReAct** [22]：将推理和行动交织在一个同步循环中，其中每一步都根据上一步的结果进行更新
- **Reflexion** [23]：从先前的错误中学习口头反思，这些反思用于指导未来的决策
- **Voyager** [24]：在 Minecraft 中利用"技能库"存储成功的行为序列，供将来复用

#### 2.1.4 行动（Action）

行动模块是将智能体的决策转化为具体操作的关键接口。我们根据行动发生的环境对行动进行归类：

**工具使用。** 智能体可以调用外部工具来扩展其能力。工具是外部函数或 API——如搜索引擎、计算器、翻译器、代码解释器——智能体可以通过生成特定格式的输出来调用它们。关键机制包括：
- LLM 根据提示中的文档选择适当的工具
- 为每个工具调用生成必要的参数
- 将工具返回的结果整合到后续推理中

**基于知识的行动。** 智能体可以直接从其内部世界知识中提取信息来回答问题，而无需调用外部工具。

**基于环境反馈的行动。** 智能体可以与物理或虚拟环境交互，如机器人操作、网页浏览或游戏玩法。环境提供观测作为反馈，智能体根据这些观测决定下一步行动。

### 2.2 能力获取

在确定了架构设计之后，下一个关键问题是：如何使智能体获得完成特定任务所需的能力？我们总结了智能体能力获取的两种主要范式：

#### 2.2.1 基于提示的方法

基于提示的方法不需要修改 LLM 的参数。它们依赖于精心设计的提示词来引导 LLM 行为：

**少量样例提示。** 在提示中提供少量输入-输出示例，使 LLM 能够通过上下文学习适应新任务。

**零样本提示。** 使用简短的指令（如 "Let's think step by step"）引导 LLM 展示所需行为，无需任何示例。

**思维链提示。** 在提示中提供中间推理步骤，鼓励 LLM 模仿该过程并生成自己的推理路径。

**自我一致性。** 生成多条推理路径并通过多数投票选择最一致的答案，以提高可靠性。

#### 2.2.2 基于微调的方法

基于微调的方法涉及使用任务特定数据更新 LLM 的参数：

**监督微调（SFT）。** 在人工标注的数据集上对预训练的 LLM 进行额外训练。例如，Toolformer [25] 使用自监督方法训练 LLM 通过嵌入在文本中的特殊标记来决定何时调用 API。

**人类反馈强化学习（RLHF）。** 使用强化学习根据人类对模型输出质量的反馈来调整 LLM 行为。InstructGPT [26] 是这种方法的一个代表性例子——它首先通过 SFT 训练模型，然后训练一个奖励模型来预测人类偏好，最后使用该奖励模型通过 PPO 来优化策略。

**偏好优化。** 直接偏好优化（DPO）[27] 是一种更简单的替代方案，它不训练单独的奖励模型，而是直接在偏好数据上优化策略。

## 3. LLM 自主智能体的应用

LLM 智能体已在多个领域展现出令人印象深刻的能力。我们将其应用分为三大类别：

### 3.1 社会科学

**社会模拟。** 基于 LLM 的智能体已被用于创建逼真的社会模拟。"生成式智能体"（Generative Agents）[28] 是一个有影响力的工作，创建了一个由 25 个 AI 智能体组成的虚拟小镇。每个智能体都有一个独特的角色、日常日程和记忆，导致涌现出社会行为——如形成关系、传播信息和组织活动。AgentSims [29] 扩展了这一概念，提供了一个用于创建和实验交互式 AI 智能体的工具箱。

**经济学。** LLM 智能体已被用于模拟经济行为，如市场参与、价格谈判和资源分配。它们可以充当经济模型中的异质性 Agent，比传统基于规则的 Agent 生成更逼真的行为模式。

**心理学。** LLM 智能体可用于探索人类行为的心理学理论。通过提示智能体假设特定的心理概况，研究人员可以在受控环境中测试假设并观察行为模式。

### 3.2 自然科学

**科学研究。** LLM 智能体可以作为不同科学领域的自主研究助手。它们能够搜索相关文献、总结发现、生成假设，并在某些情况下设计和执行实验。Coscientist [30] 证明了 LLM 智能体可以自主进行化学实验——规划合成路线、编写代码控制实验室设备、并分析结果。

**数学和逻辑推理。** LLM 智能体在解决数学问题和进行逻辑推理方面表现出色。通过工具增强，它们可以调用形式化验证器、符号计算引擎和数值解算器来补充其推理能力。

**生物学和医学。** 在生物医学领域，LLM 智能体可以协助分析基因组数据、检索科学文献、甚至协助诊断过程。它们可以在决策过程中提供循证建议。

### 3.3 工程

**软件工程。** LLM 智能体通过自动化编码、测试、调试和代码审查任务，从根本上改变了软件工程。ChatDev [31] 和 MetaGPT [32] 使用多个基于 LLM 的智能体（扮演不同角色，如产品经理、架构师、开发者和 QA 工程师）协作完成完整的软件项目。

**机器人学。** LLM 智能体可以充当将自然语言指令转化为物理动作的高级控制器。例如，Google 的 SayCan [33] 使用 LLM 将高层任务分解为原始技能，然后由低级策略执行。

**通用工具使用。** LLM 智能体在通过标准化接口使用各种 API 和软件工具方面变得非常熟练。它们可以阅读文档、选择合适的 API、构建正确的调用参数、并解释结果——使它们成为自动化复杂工作流的强大基础。

## 4. LLM 自主智能体的评估

评估 LLM 自主智能体是一个多维度的挑战。与传统模型不同——传统模型通常在特定任务上进行评估且指标定义明确——自主智能体必须根据跨多样化、通常开放式场景中的交互行为进行评估。

### 4.1 主观评估

**人类评估。** 人类评判员根据多个维度（如正确性、流畅性、有帮助性和安全性）对 Agent 的输出进行评分。虽然这提供了最直接的性能度量，但它代价高昂、耗时，并引入了评判员间的不一致性。

**LLM 评估。** 一种新兴的做法是使用强 LLM（特别是 GPT-4）作为评判员，根据特定的评分标准评估其他智能体的输出。这种方法已被 AlpacaEval、MT-Bench 和 Arena-Hard 等基准采用。关键优势是可扩展性和可复现性，但存在潜在偏见——如评判员可能偏好特定的长度或格式。

### 4.2 客观评估

**特定任务基准。** 许多基准已被提出用于评估 LLM 智能体的特定能力：
- **AgentBench** [34]：涵盖 8 个环境的综合基准——操作系统、网页浏览、数据库、知识图谱、数字卡牌游戏、横向思维谜题、家居任务、以及网页购物/浏览。它揭示了一个重要发现：LLM 作为智能体的排名与其在静态基准上的排名存在显著差异。
- **SWE-bench** [35]：从真实的 GitHub 仓库中收集了 2,294 个 issue-pull request 对，评估 Agent 解决实际软件工程问题的能力。
- **WebArena** [36]：在四个真实网站的模拟环境中评估 Agent 的网页导航和任务完成能力。
- **ToolBench**：评估 Agent 使用外部工具的能力，包含数千个真实世界的 API 和相应的指令。

## 5. 挑战与未来方向

尽管取得了显著进展，LLM 自主智能体仍面临几个关键挑战：

**角色扮演能力。** 当前的 Agent 在长期保持一致的特定角色方面仍有困难。它们可能在长对话中逐渐偏离其指定角色，或在不需要时表现出不希望的行为特征。

**长期规划和记忆。** 虽然记忆机制已被引入，但在长时间跨度内有效管理记忆并保持一致性的能力仍然有限。Agent 经常难以回忆远距离的交互或在多步骤任务中保持连贯性。

**泛化能力。** 尽管 LLM 表现出强大的跨任务泛化能力，但它们的表现可能在细粒度、领域特定任务中显著下降，特别是当任务需要专业知识或非标准推理模式时。

**幻觉。** LLM 有时会生成听起来合理但实际上不正确或虚构的内容——这种现象称为幻觉。在自主 Agent 中，幻觉可能导致复合错误，因为一个步骤中的错误输出可能传播到后续步骤并导致最终失败。

**安全问题。** 自主 Agent 可能做出有害或危险的决策。确保 Agent 操作与人类价值观一致是一个开放的研究问题。这需要设计强大的安全约束、可解释的决策过程，以及必要时的人类监督机制。

**多智能体协调。** 当多个 Agent 需要协作时，关键在于建立高效的通信协议、解决冲突、并确保整体系统可靠性。这使得个体推理的复杂性急剧放大。

**效率。** 运行自主 Agent 涉及大量的 LLM 调用，导致高昂的计算成本和延迟。如何优化资源使用，同时保持 Agent 性能，是该领域实际部署面临的关键挑战。

## 6. 结论

本文综述了 LLM 自主智能体这一新兴领域。我们将现有研究组织为三个核心维度：构建、应用和评估。我们提出了一个统一的框架，将 LLM 智能体的核心组件形式化为——大脑、记忆、规划和行动——并讨论了各种实现方法和能力获取策略。我们回顾了社会科学、自然科学和工程中广泛的应用领域，并考察了用于评估 LLM 智能体的客观和主观方法。最后，我们识别了该领域的核心挑战和值得探索的未来方向。
