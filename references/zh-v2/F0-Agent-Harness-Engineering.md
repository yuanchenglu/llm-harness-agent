## 核心贡献

提出 **ETCLOVG 七层分类法**：

| 组件 | 功能 |
|------|------|
| **E - 执行** | Agent 循环管理、轮次控制、错误恢复 |
| **T - 工具接口** | 类型化工具注册表、路由和调用管理 |
| **C - 上下文/记忆** | 上下文窗口管理、检索和优先级 |
| **L - 生命周期/编排** | 多智能体协调、任务编排 |
| **O - 可观测性** | 日志、监控、调试 |
| **V - 验证/评估** | 标准化的行为验证和评估基础设施 |
| **G - 治理/安全** | 权限控制、合规、风险管理 |

本文的三个核心主张：
1. Agent 装备层是模型与真实世界之间的独立基础设施层
2. 装备层工程需要多维度的设计和系统视角
3. 装备层质量直接决定系统可靠性和可扩展性，其影响超过模型本身的选择

**配套**：20+ 开源 Agent 装备层项目的公共目录。

**Harness 关联：** 最全面的装备层工程综述，ETCLOVG 比 A1 的六组件更细粒度。与 A1 互补——A1 重理论，F0 重工程实践。
