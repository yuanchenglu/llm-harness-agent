# DeepSeek-V2：强大、经济、高效的混合专家语言模型

**作者：** DeepSeek-AI
**论文编号：** arXiv:2405.04434v5 [cs.CL] | 2024年6月19日

---

## 摘要

我们推出 **DeepSeek-V2**，一个强大的混合专家（MoE）语言模型，具有经济训练和高效推理的特点。它拥有 236B 总参数（每 token 激活 21B），支持 128K token 的上下文长度。DeepSeek-V2 采用创新架构，包括 **多头潜在注意力（MLA）** 和 **DeepSeekMoE**。MLA 通过将键值（KV）缓存显著压缩为潜在向量来保证高效推理，而 DeepSeekMoE 通过稀疏计算以经济成本训练强模型。与 DeepSeek 67B 相比，DeepSeek-V2 实现了显著更强的性能，同时节省了 42.5% 的训练成本，减少了 93.3% 的 KV 缓存，并将最大生成吞吐量提升至 5.76 倍。我们在由 8.1T token 组成的高质量多源语料上预训练 DeepSeek-V2，并进一步执行监督微调（SFT）和强化学习（RL）以充分释放其潜力。评估结果表明，即使只有 21B 激活参数，DeepSeek-V2 及其聊天版本在开源模型中仍达到顶级性能。

## 1. 引言

过去几年，大语言模型（LLM）快速发展，让我们窥见了通用人工智能（AGI）的曙光。通常，LLM 的智能随着参数数量的增加而提升，使其在各种任务中展现出涌现能力。然而，这种提升以训练需要更大的计算资源和推理吞吐量可能下降为代价。为应对这一问题，我们推出了 DeepSeek-V2——一个通过创新 Transformer 架构实现经济训练和高效推理的强大开源 MoE 语言模型。

我们优化了 Transformer 框架中的注意力模块和前馈网络（FFN）：(1) 在注意力机制方面，多头注意力（MHA）的 KV 缓存是 LLM 推理效率的主要障碍。已有方法如分组查询注意力（GQA）和多查询注意力（MQA）在减少 KV 缓存时往往牺牲性能。为两全其美，我们提出了 **MLA**——一种配备低秩键值联合压缩的注意力机制。经验上，MLA 实现了比 MHA 更优的性能，同时显著减少了推理时的 KV 缓存。(2) 对于 FFN，我们采用 **DeepSeekMoE** 架构，该架构通过细粒度专家分割和共享专家隔离实现更高的专家专业化潜力。结合这两种技术，DeepSeek-V2 同时具备强性能、经济训练成本和高效推理吞吐量。

我们构建了包含 8.1T token 的高质量多源预训练语料。相比 DeepSeek 67B 使用的语料，本语料数据量更大（尤其是中文数据）、质量更高。我们先在完整预训练语料上预训练 DeepSeek-V2，然后收集 150 万对话会话进行 SFT，最后使用 GRPO 进行 RL 对齐。

评估结果显示，即使只有 21B 激活参数，DeepSeek-V2 在开源模型中仍达到顶级性能，成为最强的开源 MoE 语言模型。

## 2. 架构

整体上，DeepSeek-V2 仍采用 Transformer 架构，但为注意力和 FFN 设计了创新架构。

### 2.1 多头潜在注意力（MLA）：提升推理效率

标准的多头注意力（MHA）在生成时需要缓存大量 KV，成为推理瓶颈。MLA 通过低秩键值联合压缩来解决这一问题。

#### 2.1.1 标准多头注意力（预备知识）

MHA 通过三个矩阵 W_Q、W_K、W_V 从输入 h_t 生成 q_t、k_t、v_t，然后切片为 n_h 个头进行注意力计算。推理时需要缓存所有 key 和 value，每 token 需缓存 2·n_h·d_h·l 个元素。

#### 2.1.2 低秩键值联合压缩

MLA 的核心是对 key 和 value 进行低秩联合压缩：

**c_KV_t = W_DKV · h_t**（向下投影到低维潜在空间）
**k_C_t = W_UK · c_KV_t**（向上投影恢复 key）
**v_C_t = W_UV · c_KV_t**（向上投影恢复 value）

推理时 MLA 只需缓存 c_KV_t（维度 d_c），每 token 仅 d_c·l 个元素。由于 W_UK 可吸收到 W_Q 中，W_UV 可吸收到 W_O 中，我们甚至不需要显式计算 key 和 value。

为减少训练时的激活内存，我们也对 query 进行低秩压缩。

#### 2.1.3 解耦旋转位置编码

RoPE 与低秩 KV 压缩不兼容——RoPE 的位置敏感性会使 W_UK 无法吸收到 W_Q 中。解决方案：使用额外的多头 query q_R 和共享 key k_R 来承载 RoPE，即**解耦 RoPE 策略**。

#### 2.1.4 KV 缓存对比

| 注意力机制 | 每 token KV 缓存（元素数） | 能力 |
|-----------|--------------------------|------|
| MHA | 2·n_h·d_h·l | 强 |
| GQA | 2·n_g·d_h·l | 中等 |
| MQA | 2·d_h·l | 弱 |
| **MLA（本工作）** | **(d_c + d_R_h)·l ≈ 9/2·d_h·l** | **更强** |

**表 1 | 不同注意力机制的 KV 缓存对比。** DeepSeek-V2 的 KV 缓存仅相当于 2.25 组的 GQA，但性能强于 MHA。

### 2.2 DeepSeekMoE：以经济成本训练强模型

#### 2.2.1 基础架构

采用 DeepSeekMoE 架构，核心思想：细粒度专家分割（更高专家专业化、更准确知识获取）和共享专家隔离（减少路由专家间的知识冗余）。

#### 2.2.2 设备受限路由

当采用专家并行时，路由专家分布在多个设备上。设计设备受限路由机制：每个 token 的目标专家最多分布在 M 个设备上。实践中 M≥3 时性能与无限制 Top-K 路由基本一致。

#### 2.2.3 负载均衡辅助损失

设计三种辅助损失：
- **专家级均衡损失 L_ExpBal**：防止路由崩溃
- **设备级均衡损失 L_DevBal**：确保跨设备均衡计算
- **通信均衡损失 L_CommBal**：确保各设备通信均衡

#### 2.2.4 Token 丢弃策略

在训练中动态丢弃每个设备上负载过高的 token，确保计算效率，同时保留大部分训练信号。

## 3. 预训练

### 3.1 实验设置

**数据构建：** 8.1T token 的高质量多源语料，相比 DeepSeek 67B 数据量更大、质量更高，中文数据比例从 28.6% 提升到 31.3%。

**超参数：** 236B 总参数，21B 激活参数，128K 上下文。MoE 层配置：1 个共享专家 + 64 个路由专家，每 token 激活 8 个路由专家。

| 超参数 | DeepSeek-V2 |
|--------|------------|
| 隐藏维度 | 5120 |
| FFN 中间维度 | 12288 |
| 注意力头数 | 128 |
| KV 压缩维度 d_c | 512 |
| 层数 | 80 |
| 训练 token 数 | 8.1T |

**基础设施：** 基于 HAI-LLM 框架，使用 8 路张量并行 + 64 路专家并行 + ZeRO1，配备 200 块 H800 GPU。

**长上下文扩展：** 使用 YaRN 方法，在额外 1000 步训练中将上下文从 4K 扩展到 128K。

### 3.2 评估

**基座模型评估：** DeepSeek-V2 在 MMLU（79.0%）、ARC-Challenge、HellaSwag 等基准上达到开源模型顶级水平，21B 激活参数远超同激活量模型，甚至超越许多更大模型。

**训练与推理效率：** 相比 DeepSeek 67B：
- 训练成本降低 42.5%
- KV 缓存减少 93.3%
- 最大生成吞吐量提升 5.76 倍

## 4. 对齐

### 4.1 监督微调（SFT）

收集 150 万对话会话，涵盖数学、代码、写作、推理、安全等领域。使用 AdamW 优化器，学习率 1e-5，训练 2 个 epoch。

### 4.2 强化学习（RL）

使用 GRPO 算法进一步对齐人类偏好。在数学、代码和推理数据上训练，生成立即反馈式奖励模型。

### 4.3 评估结果

| 基准 | DeepSeek-V2 Chat (SFT) | DeepSeek-V2 Chat (RL) |
|------|----------------------|----------------------|
| MMLU (5-shot) | 79.3% | 79.0% |
| GSM8K (8-shot) | 86.2% | 88.9% |
| MATH (4-shot) | 40.7% | 43.7% |
| HumanEval (0-shot) | 74.4% | 73.8% |
| AlpacaEval 2.0 (LC) | 20.5% | **38.9%** |
| MT-Bench | 8.80 | **8.97** |
| AlignBench | 7.29 | **7.91** |

**表 2 | 对齐评估结果。** RL 在开放式生成任务上带来显著提升。

### 4.4 讨论

- **Scaling RL 数据**：RL 数据量从 13K 扩展到 257K 时，对话性能持续提升。
- **Scaling 奖励模型**：设置合理的 KL 惩罚系数可在利用奖励模型和保持生成多样性间取得平衡。
- **数学任务上的 RL 扩展**：在数学推理上，RL 带来的收益随训练步数增加而递增。

---

## 5. 结论、局限与未来工作

DeepSeek-V2 通过 MLA 和 DeepSeekMoE 实现强大性能、经济训练和高效推理。局限包括：
- 在 MMLU 等多项选择题基准上受限于注意力参数容量
- 部署需要多 GPU，小规模部署场景受限

我们已开源 DeepSeek-V2-Lite（15.7B 总参数，2.4B 激活参数），供社区研究 MLA 和 DeepSeekMoE。
