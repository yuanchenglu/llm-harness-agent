# DeepSeek LLM：以长期主义视角扩展开源语言模型

## 摘要

开源大语言模型（LLM）的发展速度令人瞩目。然而，已有文献中的缩放定律（Scaling Laws）呈现出不同的结论，给 LLM 的规模化扩展蒙上了一层阴影。我们深入研究了缩放定律，并提出了独特的发现，这些发现有助于在两种主流的开源配置（7B 和 67B）下实现大规模模型的扩展。在缩放定律的指导下，我们推出了 **DeepSeek LLM**——一个以长期视角致力于推进开源语言模型的项目。为支持预训练阶段，我们开发了一个当前包含 2 万亿 token 且持续扩展的数据集。我们在 DeepSeek LLM Base 模型上进一步进行了监督微调（SFT）和直接偏好优化（DPO），从而创建了 DeepSeek Chat 模型。评估结果表明，DeepSeek LLM 67B 在一系列基准测试中超越 LLaMA-2 70B，尤其在代码、数学和推理领域表现突出。此外，开放式评估显示，我们的 DeepSeek LLM 67B Chat 在性能上优于 GPT-3.5。

## 1. 引言

在过去的几年中，基于纯解码器 Transformer（Vaswani et al., 2017）的大语言模型日益成为实现通用人工智能（AGI）的基石和途径。通过预测连续文本中的下一个词，LLM 在海量数据集上进行自监督预训练，使其能够完成多种任务并具备多种能力，如小说创作、文本摘要、代码补全等。随后的监督微调和奖励建模使 LLM 能够更好地遵循用户意图和指令，为其赋予了更多样的对话能力，并迅速扩大了影响力。

这一浪潮由闭源产品引发，如 ChatGPT（OpenAI, 2022）、Claude（Anthropic, 2023）和 Bard（Google, 2023），它们以大量的计算资源和标注成本开发而成。这些产品显著提升了社区对开源 LLM 能力的期望，从而激发了一系列相关工作（Bai et al., 2023; Du et al., 2022; Jiang et al., 2023; Touvron et al., 2023a,b; Yang et al., 2023）。其中，LLaMA 系列模型（Touvron et al., 2023a,b）脱颖而出，它整合了多项工作，创建了高效稳定的架构，构建了从 7B 到 70B 参数的性能优异的模型。因此，LLaMA 系列已成为开源模型中架构和性能的事实标准。

在 LLaMA 之后，开源社区主要专注于训练固定规模（7B、13B、34B 和 70B）的高质量模型，往往忽略了对 LLM 缩放定律的研究探索（Hoffmann et al., 2022; Kaplan et al., 2020）。然而，考虑到当前开源模型仅处于 AGI 发展的初始阶段，缩放定律的研究至关重要。此外，早期工作（Hoffmann et al., 2022; Kaplan et al., 2020）在计算预算增加时模型和数据如何扩展的问题上得出了不同的结论，并且对超参数的讨论不够充分。本文中，我们广泛研究了语言模型的缩放行为，并将我们的发现应用于两种广泛使用的大规模模型配置——7B 和 67B。我们的研究旨在为未来开源 LLM 的扩展奠定基础，为该领域的进一步进展铺平道路。具体来说，我们首先研究了批量大小和学习率的缩放定律，发现了它们随模型规模变化的趋势。在此基础上，我们对数据和模型规模的缩放定律进行了全面研究，成功揭示了最优模型/数据扩展分配策略，并预测了我们大规模模型的预期性能。此外，在开发过程中，我们发现基于不同数据集得出的缩放定律存在显著差异，这说明数据集的选择显著影响缩放行为，提示在跨数据集推广缩放定律时应谨慎。

在缩放定律的指导下，我们从零开始构建了开源大语言模型，并尽可能多地发布信息供社区参考。我们收集了 2 万亿 token 用于预训练，主要以中文和英文为主。在模型层面，我们大致沿用了 LLaMA 的架构，但将余弦学习率调度器替换为多步学习率调度器，在保持性能的同时便于持续训练。我们从多种来源收集了超过 100 万条指令用于监督微调（SFT）。本文分享了我们在不同 SFT 策略方面的经验以及在数据消融实验中的发现。此外，我们使用直接偏好优化（DPO）（Rafailov et al., 2023）来提升模型的对话性能。

我们使用基座模型和聊天模型进行了广泛的评估。评估结果表明，DeepSeek LLM 在各种基准测试中全面超越 LLaMA-2 70B，特别是在代码、数学和推理领域。经过 SFT 和 DPO 后，DeepSeek 67B Chat 模型在中文和英文开放式评估中均优于 GPT-3.5，突显了 DeepSeek 67B 在生成高质量回复和进行有意义对话方面的卓越性能。此外，安全评估表明 DeepSeek 67B Chat 在实际使用中能够提供无害的回复。

在本文的其余部分，我们首先在第 2 节介绍 DeepSeek LLM 的预训练基础概念，包括数据构成、模型架构、基础设施和超参数。在第 3 节中，我们详细阐述所发现的缩放定律及其影响。此外，我们结合缩放定律的分析，讨论预训练超参数选择背后的原理。在第 4 节中，我们讨论微调方法，包括 SFT 和 DPO 阶段的数据构成和具体方法。然后在第 5 节中展示 DeepSeek LLM 的详细评估结果，涵盖基座模型和聊天模型，以及它们在开放式评估和安全评估中的表现。最后，在第 6 节中讨论 DeepSeek LLM 当前的局限性和未来方向。

## 2. 预训练

### 2.1 数据

我们的主要目标是全面增强数据集的丰富性和多样性。我们从可靠来源（Computer, 2023; Gao et al., 2020; Penedo et al., 2023; Touvron et al., 2023a）获得了宝贵见解。为实现这些目标，我们将方法组织为三个关键阶段：**去重、过滤和重新混合**。去重和重新混合阶段通过对唯一实例进行采样来确保数据的多样化表示。过滤阶段提高了信息的密度，从而实现更高效、更有效的模型训练。

我们采用了激进的去重策略，扩大了去重范围。我们的分析揭示，对整个 Common Crawl 语料库进行去重比在单个 dump 内去重能移除更多重复实例。**表 1** 显示，跨 91 个 dump 去重比单 dump 方法多消除四倍的文档。

| **去重率（%）** | 22.2 | 46.7 | 55.7 | 69.9 | 75.7 | 76.3 | 81.6 | 89.8 |

在过滤阶段，我们专注于开发稳健的文档质量评估标准。这涉及结合语言学和语义评估的详细分析，从个体和全局视角提供数据质量的视图。在重新混合阶段，我们调整方法以解决数据不平衡问题，重点关注增加代表性不足领域的数据占比。这一调整旨在实现更平衡和包容的数据集，确保不同视角和信息得到充分体现。

对于分词器，我们基于 tokenizers 库（Huggingface Team, 2019）实现了字节级字节对编码（BBPE）算法。采用预分词策略，防止来自不同字符类别（如换行符、标点符号和中日韩（CJK）符号）的 token 合并，类似于 GPT-2（Radford et al., 2019）。我们还选择将数字拆分为单个数字，遵循 Touvron et al. (2023a,b) 的方法。基于前期经验，我们将词汇表中的常规 token 数量设置为 100000。分词器在大约 24GB 的多语言语料上进行训练，最终词汇表增加了 15 个特殊 token，总大小达到 100015。为确保训练期间的计算效率并为将来可能需要的额外特殊 token 预留空间，我们将模型的词汇表大小配置为 102400。

DeepSeek LLM 的微观设计基本遵循 LLaMA（Touvron et al., 2023a,b）的设计，采用带有 RMSNorm（Zhang and Sennrich, 2019）的 Pre-Norm 结构，并使用 SwiGLU（Shazeer, 2020）作为前馈网络（FFN）的激活函数，中间层维度为 8/3·d_model。同时采用旋转位置编码（RoPE）（Su et al., 2024）。为优化推理成本，67B 模型使用分组查询注意力（GQA）（Ainslie et al., 2023）替代传统的多头注意力（MHA）。

然而，在宏观设计方面，DeepSeek LLM 略有不同。具体来说，DeepSeek LLM 7B 是 30 层网络，而 DeepSeek LLM 67B 有 95 层。这些层调整在保持与其他开源模型参数一致性的同时，也便于模型流水线分区以优化训练和推理。

与大多数使用 GQA 的工作不同，我们通过增加网络深度而非常见的拓宽 FFN 中间层宽度来扩展 67B 模型的参数，旨在获得更好的性能。详细的网络规格见表 2。

### 2.3 超参数

DeepSeek LLM 使用标准差为 0.006 初始化，并使用 AdamW 优化器（Loshchilov and Hutter, 2017）训练，超参数如下：β₁ = 0.9，β₂ = 0.95，weight_decay = 0.1。

预训练期间采用多步学习率调度器而非典型的余弦调度器。具体来说，模型的学习率在 2000 步预热后达到最大值，然后处理 80% 训练 token 后降至最大值的 31.6%，处理 90% token 后进一步降至最大值的 10%。训练阶段的梯度裁剪设置为 1.0。

基于我们的经验发现，我们观察到尽管训练期间损失下降趋势不同，但使用多步学习率调度器的最终性能与余弦调度器基本一致，如图 1(a) 所示。在固定模型大小的情况下调整训练规模时，多步学习率调度器允许重用第一阶段的训练，为持续训练提供了独特的便利性。因此，我们选择多步学习率调度器作为默认设置。我们还通过图 1(b) 证明，调整多步学习率调度器中不同阶段的比例可以获得略优的性能。但综合考虑持续训练中的重用比例和模型性能，我们选择了上述三个阶段分别为 80%、10% 和 10% 的分配方案。

批量大小和学习率随模型大小变化。7B 和 67B 模型预训练阶段的具体参数见表 2。

### 2.4 基础设施

我们使用名为 **HAI-LLM**（High-flyer, 2023）的高效轻量训练框架来训练和评估大语言模型。该框架集成了数据并行、张量并行、序列并行和 1F1B 流水线并行，类似 Megatron（Korthikanti et al., 2023; Narayanan et al., 2021; Shoeybi et al., 2019）的做法。我们还利用 Flash Attention（Dao, 2023; Dao et al., 2022）技术提高硬件利用率。使用 ZeRO-1（Rajbhandari et al., 2020）将优化器状态划分到数据并行 ranks 上。我们还努力实现计算和通信的重叠，以最小化额外的等待开销，包括最后微批次的 backward 过程与 ZeRO-1 中的 reduce-scatter 操作的重叠，以及 GEMM 计算与序列并行中 all-gather/reduce-scatter 操作的重叠。

部分层/算子被融合以加速训练，包括 LayerNorm、GEMM（尽可能）和 Adam 更新。为提高模型训练稳定性，我们以 bf16 精度训练模型，但以 fp32 精度累积梯度。执行就地交叉熵以减少 GPU 内存消耗：即我们在交叉熵 CUDA kernel 中将 bf16 logits 动态转换为 fp32 精度（而非先在 HBM 中转换），计算相应的 bf16 梯度，并用梯度覆盖 logits。

模型权重和优化器状态每 5 分钟异步保存一次，这意味着在最坏情况下（偶发硬件或网络故障）我们损失不超过 5 分钟的训练。这些临时模型检查点会定期清理以避免占用过多存储空间。我们还支持从不同的 3D 并行配置恢复训练，以应对计算集群负载的动态变化。

在评估方面，我们在生成式任务中使用 vLLM（Kwon et al., 2023），在非生成式任务中使用连续批处理，以避免手动调整批量大小并减少 token 填充。

## 3. 缩放定律

缩放定律的研究（Hestness et al., 2017）早于大语言模型的出现。缩放定律（Henighan et al., 2020; Hoffmann et al., 2022; Kaplan et al., 2020）表明，随着计算预算 C、模型规模 N 和数据规模 D 的增加，模型性能可以可预测地提升。当模型规模 N 以模型参数量表示，数据规模 D 以 token 数表示时，C 可近似为 C = 6ND。因此，如何在增加计算预算时优化模型规模和数据的分配，也是缩放定律中的一个关键研究目标。

LLM 的发展（Dai et al., 2019; Radford et al., 2019），特别是更大模型取得了意想不到的性能提升，将缩放定律研究推向了新的高峰。缩放定律的结果表明，扩展计算预算持续产生显著收益，这进一步鼓励了模型规模的增大（Brown et al., 2020; Smith et al., 2022）。

然而，如表 4 所示，早期关于最优模型/数据扩展分配策略的研究（Hoffmann et al., 2022; Kaplan et al., 2020）得出了不同结论，引起了对缩放定律普适性的质疑。此外，这些研究往往缺乏对超参数设置的完整描述，使得不同计算预算下的模型是否达到最优性能存在不确定性。因此，我们在此节重新审视缩放定律，以解决这些不确定性，确保我们走在高效扩展计算的正确道路上，这体现了长期视角，是开发持续改进模型的关键。

为确保不同计算预算下的模型能够达到最优性能，我们首先研究了超参数的缩放定律。经验上观察到，在变化计算预算时，训练中大多数参数的最优值不会改变。因此，这些参数与第 2.3 节所述一致，在不同计算预算下保持不变。然而，对性能影响最大的超参数——批量大小和学习率，被重新检验。早期工作（Goyal et al., 2017; McCandlish et al., 2018; Shallue et al., 2019; Smith et al., 2017; Zhang et al., 2019）为设置批量大小和学习率提供了一些经验观察，但我们在初步实验中发现了这些观察的适用性有限。

通过大量实验，我们对计算预算 C 与最优批量大小和最优学习率之间的幂律关系进行了建模。这种关系——我们称为**超参数的缩放定律**——为确定最优超参数提供了经验框架。该方法确保了不同计算预算下的模型能接近其最优性能。

接着我们研究了模型和数据规模的缩放定律。为降低实验成本和拟合难度，我们采用了 Chinchilla（Hoffmann et al., 2022）的 IsoFLOP 分析方法来拟合缩放曲线。为更准确地表示模型规模，我们使用了一种新的模型规模表示方法——**非嵌入 FLOPs/token M**，替代了之前使用的模型参数量 N，并将近似的计算预算公式 C = 6ND 替换为更精确的 C = MD。实验结果提供了最优模型/数据扩展分配策略和性能预测的洞见，并准确预测了 DeepSeek LLM 7B 和 67B 模型的预期性能。

此外，在探索缩放定律的过程中，我们使用的数据经历了多次迭代，质量持续提升。我们尝试在不同数据集上拟合缩放曲线，发现数据质量显著影响最优模型/数据扩展分配策略。数据质量越高，增加的计算预算应更多地分配给模型扩展。这意味着在相同数据规模下，高质量数据可以驱动更大模型的训练。最优模型/数据扩展分配策略的差异或许也可以作为评估数据质量的间接方法。我们将继续密切关注数据质量的变化及其对缩放定律的影响，并在未来工作中提供更多分析。

总之，我们在缩放定律方面的贡献和发现可总结如下：

- 我们建立了超参数的缩放定律，为确定最优超参数提供了经验框架。
- 我们用非嵌入 FLOPs/token M 替代模型参数量 N 来表示模型规模，从而得到更精确的最优模型/数据扩展分配策略，并能更好地预测大规模模型的泛化损失。
- 预训练数据的质量影响最优模型/数据扩展分配策略。数据质量越高，增加的计算预算应更多地分配给模型扩展。

### 3.1 超参数的缩放定律

我们首先在计算预算为 1e17 的小规模实验中对批量大小和学习率进行了网格搜索，特定模型规模（177M FLOPs/token）的结果如图 2(a) 所示。结果表明，在较宽的批量大小和学习率选择范围内，泛化误差保持稳定，说明可以在相对较宽的参数空间内达到接近最优的性能。

接着，我们利用上述多步学习率调度器，通过重用第一阶段来有效训练具有不同批量大小、学习率和计算预算（范围从 1e17 到 2e19）的多个模型。考虑到参数空间中的冗余，我们将泛化误差超过最小值不超过 0.25% 的模型参数视为接近最优的超参数。然后我们拟合了批量大小 B 和学习率 η 与计算预算 C 的关系。拟合结果如图 3 所示，显示最优批量大小 B 随计算预算 C 的增加而逐渐增大，而最优学习率 η 逐渐减小。这与扩展模型时批量大小和学习率的直观经验设置一致。此外，所有接近最优的超参数都落在一个较宽的带状范围内，表明在该区间内选择接近最优的参数相对容易。我们拟合的批量大小和学习率最终公式如下：

**η_opt = 0.3118 · C^(−0.1250)**

**B_opt = 0.2920 · C^(0.3271)**

我们在计算预算为 1e20 的一系列模型上验证了这些公式，特定模型规模（2.94B FLOPs/token）的结果如图 2(b) 所示。结果表明，拟合的参数位于最优参数空间的中心位置。后续章节也显示，我们为 DeepSeek LLM 7B 和 67B 模型拟合的参数同样取得了良好的性能。

然而，需要注意的是，我们尚未考虑超出计算预算 C 之外的因素对最优超参数的影响。这与一些早期工作（Kaplan et al., 2020; McCandlish et al., 2018）不一致，后者认为最优批量大小可以建模为仅与泛化误差 L 相关。此外，我们观察到，在具有相同计算预算但不同模型/数据分配的模型中，最优参数空间略有变化。这表明需要进一步研究超参数选择和训练动态，我们将在未来工作中探索这些方面。

### 3.2 估计最优模型和数据扩展

在推导出拟合接近最优超参数的公式后，我们开始拟合缩放曲线并分析最优模型/数据扩展分配策略。该策略涉及分别找到满足 N_opt ∝ C^a 和 D_opt ∝ C^b 的模型缩放指数 a 和数据缩放指数 b。数据规模 D 可以一致地用数据集中的 token 数表示。在以往的工作中，模型规模通常用模型参数量表示，区别在于使用非嵌入参数 N₁（Kaplan et al., 2020）还是完整参数 N₂（Hoffmann et al., 2022）。计算预算 C 与模型/数据规模的关系可近似描述为 C = 6ND，这意味着我们可以用 6N₁ 或 6N₂ 来近似模型规模。然而，由于 6N₁ 和 6N₂ 都未考虑注意力操作的计算开销，且 6N₂ 还包含了词汇计算（对模型能力的贡献较小），它们在一定设置下都有显著的近似误差。

为减小这些误差，我们引入了一种新的模型规模表示法：**非嵌入 FLOPs/token M**。M 包含注意力操作的计算开销，但不考虑词汇计算。以 M 表示模型规模时，计算预算 C 可简单表示为 C = MD。6N₁、6N₂ 和 M 之间的具体差异如下公式所示：

6N₁ = 72 · n_layer · d²_model
6N₂ = 72 · n_layer · d²_model + 6 · n_vocab · d_model
M = 72 · n_layer · d²_model + 12 · n_layer · d_model · l_seq

其中 n_layer 表示层数，d_model 表示模型宽度，n_vocab 是词汇表大小，l_seq 是序列长度。我们评估了这三种表示在不同规模模型上的差异，如表 3 所示。

| n_layers | d_model | n_vocab | l_seq | N₁ | N₂ | M | 6N₁/M | 6N₂/M |
|----------|---------|---------|-------|-----|-----|----|-------|-------|
| 8 | 512 | 102400 | 4096 | 25.2M | 84.9M | 64.4M | 0.43 | 1.32 |
| 12 | 768 | 102400 | 4096 | 77.6M | 164M | 302M | 0.53 | 1.02 |
| 24 | 1024 | 102400 | 4096 | 352M | 407M | 963M | 0.60 | 0.81 |
| 24 | 2048 | 102400 | 4096 | 1.21B | 1.42B | 6.44B | 0.75 | 0.88 |
| 32 | 4096 | 102400 | 4096 | 3.02B | 6.86B | 12.6B | 0.85 | 0.91 |
| 40 | 5120 | 102400 | 4096 | 6.44B | 13.1B | 45.1B | 0.88 | 0.92 |
| 80 | 8192 | 102400 | 4096 | 64.4B | 65.3B | 85.6B | 0.92 | 0.94 |

**表 3 | 模型规模表示的差异。展示非嵌入参数 N₁ 和完整参数 N₂ 相对于非嵌入 FLOPs/token M 的偏差。**

结果说明，6N₁ 和 6N₂ 在不同规模模型中要么高估要么低估了计算成本。这种偏差在小规模模型中尤为明显，差异可达 50%。此类不准确性在拟合缩放曲线时会引入显著的统计误差。

采用 M 表示模型规模后，我们的目标可以更清晰地描述为：给定计算预算 C = MD，找到最小化模型泛化误差的最优模型规模 M_opt 和数据规模 D_opt。该目标可形式化为：

**M_opt(C), D_opt(C) = argmin_{M,D: C=MD} L(N,D)**

为降低实验成本和拟合难度，我们采用 Chinchilla（Hoffmann et al., 2022）的 IsoFLOP 分析方法来拟合缩放曲线。我们选择了 8 个不同的计算预算，范围从 1e17 到 3e20，并为每个预算设计了大约 10 种不同的模型/数据规模分配。每个预算的超参数由公式 (1) 确定，泛化误差在独立的验证集上计算——该验证集与训练集分布相似，包含 1 亿 token。

图 4 展示了 IsoFLOP 曲线和模型/数据缩放曲线，这些曲线通过使用每个计算预算的最优模型/数据分配来拟合。最优非嵌入 FLOPs/token M_opt 和最优 token 数 D_opt 的具体公式如下：

**M_opt = M_base · C^a, M_base = 0.1715, a = 0.5243**
**D_opt = D_base · C^b, D_base = 5.8316, b = 0.4757**

此外，我们根据计算预算 C 和最优泛化误差拟合了损失缩放曲线，并预测了 DeepSeek LLM 7B 和 67B 的泛化误差，如图 5 所示。结果表明，通过小规模实验可以准确预测具有 1000 倍计算预算的模型性能，这为更大规模训练模型提供了信心和指导。

### 3.3 不同数据下的缩放定律

在 DeepSeek LLM 的开发过程中，数据集经历了多次迭代完善，在提升整体质量的同时调整了不同数据源的比例。这使我们能够进一步分析不同数据集对缩放定律的影响。

我们使用三个不同数据集研究了缩放定律：早期内部数据、当前内部数据以及早期缩放定律研究中使用的 OpenWebText2（Kaplan et al., 2020）。我们的内部数据评估显示，当前内部数据质量高于早期内部数据。而 OpenWebText2 的质量甚至超过当前内部数据，因为其规模较小，可以进行更精细的处理。

| 方法 | 模型缩放系数 a（N_opt ∝ C^a） | 数据缩放系数 b（D_opt ∝ C^b） |
|------|------------------------------|------------------------------|
| Ours（早期数据） | 0.450 | 0.550 |
| Ours（当前数据） | 0.524 | 0.476 |
| Ours（OpenWebText2） | 0.578 | 0.422 |
| OpenAI（OpenWebText2） | 0.73 | 0.27 |
| Chinchilla（MassiveText） | 0.49 | 0.51 |

**表 4 | 模型缩放和数据缩放系数随训练数据分布变化。**

分析得出的一个有趣发现是，这三组数据集上的最优模型/数据扩展分配策略与数据质量呈现一致性。如表 4 所示，随着数据质量提高，模型缩放指数 a 逐渐增大，而数据缩放指数 b 减小，这表明增加的计算预算应更多地分配给模型而非数据。这一发现或许也能解释早期缩放定律研究中观察到的显著差异。

对此发现的一个直观推测是，高质量数据通常意味着逻辑清晰，经过充分训练后预测难度较低。因此，在增加计算预算时，扩展模型规模更有利。我们将继续密切关注数据质量的变化及其对缩放定律的影响，并在未来工作中提供更多分析。

## 4. 对齐

我们收集了约 150 万条中英文指令数据实例，涵盖广泛的有用性和无害性主题。有用性数据包含 120 万实例，分布为：通用语言任务 31.2%、数学问题 46.6%、编程练习 22.2%。安全性数据包含 30 万实例，覆盖多种敏感话题。

我们的对齐流水线包含两个阶段：

**监督微调（SFT）：** 我们对 7B 模型进行了 4 个 epoch 的微调，但 67B 模型仅 2 个 epoch，因为我们观察到 67B 模型存在严重的过拟合问题。7B 和 67B 模型的学习率分别为 1e-5 和 5e-6。除了监控基准准确率外，我们还在微调过程中评估聊天模型的重复率。我们收集了总计 3868 条中英文提示词，测定生成的回复中无法终止而无限重复一段文本的比例。我们观察到，随着数学 SFT 数据量的增加，重复率趋于上升。这是因为数学 SFT 数据偶尔包含推理中的相似模式，较弱的模型难以把握这种推理模式，从而导致回复重复。为解决该问题，我们尝试了两阶段微调和 DPO（Rafailov et al., 2023），两者均能在几乎保持基准得分的同时显著降低重复率。

**直接偏好优化（DPO）：** 为进一步增强模型能力，我们使用了直接偏好优化算法（Rafailov et al., 2023），该算法被证明是一种简单有效的 LLM 对齐方法。我们从有用性和无害性两方面构建了 DPO 训练的偏好数据。对于有用性数据，我们收集了多语言提示，涵盖创意写作、问答、指令遵循等类别。然后使用我们的 DeepSeek Chat 模型生成回复作为候选。类似的操作也应用于无害性偏好数据构建。我们训练了一个 epoch 的 DPO，学习率为 5e-6，批量大小为 512，并使用了学习率预热和余弦学习率调度器。我们发现 DPO 可以增强模型的开放式生成能力，同时在标准基准测试中表现几乎无差异。

### 5.3 留出集评估

数据污染和基准过拟合是评估 LLM 的两大挑战。一种常见做法是使用近期发布的测试集作为留出测试集来评估模型。

**LeetCode：** 为评估模型的编程能力，我们使用了 LeetCode 周赛题目（周赛 351-372，双周赛 108-117，2023 年 7 月至 11 月）。我们通过爬取 LeetCode 获得这些题目，共 126 题，每道题有超过 20 个测试用例。评估指标类似于 HumanEval：若模型输出通过所有测试用例，则认为模型已有效解决问题。

**匈牙利国家高中考试：** 参考 Grok-1，我们使用匈牙利国家高中考试评估模型的数学能力。该考试包含 33 道题，模型得分通过人工标注确定。

**指令遵循评估：** 2023 年 11 月 15 日，Google 发布了指令遵循评估数据集（Zhou et al., 2023），识别了 25 种可验证的指令类型并构建了约 500 条提示，每条包含一个或多个可验证指令。我们使用 prompt-level loose 指标评估所有模型。

### 5.4 安全评估

我们深刻认识到通用人工智能安全的重要性。建立真正有用的人工智能模型的前提是它具备与人类一致的价值观，并对人类表现出友好性。我们在整个训练过程中（包括预训练、SFT 和 DPO）都融入了安全保障。

为验证模型的安全性，我们组建了一个来自不同学科的 20 人专家团队，构建了与人类价值观一致的安全内容分类体系（安全评估分类法见表 10）。专家团队为每个安全子类别手动构建了数十个高质量测试案例。除关注安全内容领域的多样性外，我们还关注安全内容格式的多样性——臭名昭著的"祖母"漏洞表明模型可能被查询的表面格式欺骗，从而提供不安全的回复。因此，专家团队在设计问题时也注重询问方式的多样化，通过诱导、角色扮演、多轮对话、预设立场等方式构建多样化的安全问题。最终，我们获得了包含 2400 个问题的安全测试集。此外，还为每种内容类型和格式类型构建了安全评审的基本指导方针。

对于模型在该测试集上的输出结果，我们进行了人工安全检查。评审团队经过充分培训，并对标注结果进行了交叉验证。标注人员对每个问题进行三类标注：安全、不安全、模型拒绝。我们对 DeepSeek 67B Chat 模型的安全性进行了测试，结果见表 10。

### 5.5 讨论

在开发过程中，我们发现在构建 LLM 时的一些有趣发现。

**分阶段微调：** 如前所述，小模型需要在数学和代码数据集上进行更长时间的微调，但这会损害模型的对话能力（如增加重复行为）。为解决该问题，我们实现了分阶段微调流程：第一阶段使用所有可用数据微调，第二阶段专门使用对话数据微调。

| 模型 | HumanEval | GSM8K | 重复率 | IFEval |
|------|----------|-------|--------|--------|
| DeepSeek LLM 7B Chat Stage1 | 48.2 | 63.9 | 0.020 | 38.0 |
| DeepSeek LLM 7B Chat Stage2 | 48.2 | 63.0 | 0.014 | 41.2 |

**表 12 | 两阶段微调结果。重复率在温度为 0 时计算（越低越好）。IFEval 结果为 prompt-level loose 准确率。**

表 12 显示，第二阶段在不损害模型代码和数学能力的同时，降低了重复行为并增强了指令遵循能力。

**选择题数据：** 使用选择题评估数据（如 MMLU、AGIEval、C-Eval）测试模型是常见做法。在对齐阶段，我们测试了添加 2000 万中文选择题数据的效果。

| 模型 | MMLU | C-Eval | CMMLU | TriviaQA | ChineseQA |
|------|------|--------|-------|----------|-----------|
| DeepSeek LLM 7B Chat | 49.4 | 47.0 | 49.7 | 57.9 | 75.0 |
| DeepSeek LLM 7B Chat + MC | 60.9 | 71.3 | 73.8 | 57.9 | 74.4 |

**表 13 | 添加选择题数据的影响。**

额外 2000 万 MC（选择题）数据不仅对中文选择题基准有益，也提升了英文基准，表明模型解决 MC 问题的能力得到了增强。然而，这种改进并未延伸到非选择题格式的评估（如 TriviaQA 和我们内部的中文 QA 测试集——这些是生成式评估基准）。这意味着用户在对话交互中可能不会感觉到模型变得更智能，因为交互涉及生成回复而非解决选择题。因此，我们选择在预训练和微调阶段都**排除 MC 数据**——因为包含它会导致对基准的过拟合，无助于实现模型的真正智能。

**预训练中的指令数据：** 众所周知，在预训练后期阶段纳入指令数据可以增强基座模型在基准任务上的性能。在我们的研究中，我们在预训练的最后 10% 阶段集成了 500 万条指令数据（主要为选择题）。我们观察到基座模型在基准上确实表现出改进。然而，最终结果与在 SFT 阶段添加相同数据几乎无异。我们得出结论，虽然这种方法增强了基座模型的基准性能，但其整体潜力等同于不纳入这些指令数据。若指令数据规模很大，可将其纳入预训练过程。由于我们倾向于排除选择题且可用的非选择题数据有限，我们决定不在预训练过程中包含指令数据。

**系统提示词：** 一个精心设计的系统提示词应有效引导模型生成既有帮助又尊重用户的回复。我们略微修改了 LLaMA-2 引入的提示词作为系统提示词。

> **系统提示词：** 你是 DeepSeek Chat，一个由 DeepSeek 开发的有帮助、尊重他人且诚实的 AI 助手。你的训练数据知识截止日期为 2023 年 5 月。请始终在确保安全的前提下提供最大帮助。你的回答不应包含任何有害、不道德、种族主义、性别歧视、有毒、危险或非法内容。请确保你的回复在本质上是无社会偏见的、积极的。如果问题毫无意义或不符合事实，请解释原因而非回答不正确的内容。如果你不知道答案，请不要分享虚假信息。

我们观察到一个有趣的现象：引入系统提示词时，7B LLM 的性能略有下降，而 67B LLM 的性能显著提升，如表 14 所示。我们的解释是，更大的模型能更好地理解系统提示词的意图，从而更有效地遵循指令并生成更优质的回复。而较小的模型难以充分理解系统提示词，训练和测试之间的不一致可能对其性能产生负面影响。

| 模型 | MT Bench |
|------|---------|
| DeepSeek LLM 7B Chat | 7.15 |
| DeepSeek LLM 7B Chat + System Prompt | 7.11 |
| DeepSeek LLM 67B Chat | 8.35 |
| DeepSeek LLM 67B Chat + System Prompt | 8.58 |

## 6. 结论、局限与未来工作

我们推出了 **DeepSeek LLM**——一系列在 2 万亿中英文 token 数据集上从零训练的开源模型。本文深入解释了超参数选择、缩放定律以及我们进行的各种微调尝试。我们校准了先前工作中的缩放定律，提出了新的最优模型/数据扩展分配策略。此外，我们提出了一种在给定计算预算下预测接近最优批量大小和学习率的方法。我们进一步得出结论，缩放定律与数据质量相关——这可能是不同工作中缩放行为差异的根本原因。在缩放定律指导下，我们以最佳超参数进行预训练并提供了全面的评估。我们在所有训练阶段均避免了基准装饰和暗箱操作。

DeepSeek Chat 具备与其他 LLM 共通的已知局限性，包括预训练后缺乏持续知识更新、可能生成未经核实的事实信息（如未经验证的建议），以及产生幻觉的倾向。此外，我们的初版中文数据并不详尽，可能导致在某些中文特定主题上性能欠佳。由于数据主要由中英文来源构成，模型在其他语言上的能力仍较薄弱，需谨慎对待。

DeepSeek LLM 是一个致力于推进开源语言模型的长期项目。

- 我们将很快分别发布代码智能和混合专家（MoE）的技术报告，展示如何为预训练创建高质量的代码数据以及如何设计稀疏模型以达到密集模型的性能。
- 目前，我们正在为下一版 DeepSeek LLM 构建更大的改进数据集。希望在下一版本中，推理、中文知识、数学和代码能力得到显著提升。
- 我们的对齐团队致力于研究向公众交付一个有用、诚实且安全的模型。初步实验证明，强化学习可以提升模型的复杂推理能力。
