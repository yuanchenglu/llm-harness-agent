# 0-3 证据规范 Evidence Standard

## 1. 证据等级

| 等级 | 含义 |
|---|---|
| S0 | 官方源码 / 官方 config / 官方模型仓库直接确认 |
| S1 | 官方模型卡 / 官方 API 文档直接确认 |
| A | 官方技术报告确认 |
| B | 可信第三方资料 |
| C | 工程推论，必须标注，不得当作官方事实 |

## 2. 文档使用原则

1. 写正式产品战略时，优先引用 `2-证据增强终稿`。
2. 需要细节推理过程时，回查 `1-原始阶段文档全集`。
3. 需要证明“没有丢文件”时，查看 `99-原始压缩包备份`。
4. 任何 Harness 设计结论都应能追溯到：
   - V4 官方源码；
   - 官方 config；
   - 官方模型卡；
   - 官方 API Docs；
   - 或明确标注为工程推论。

## 3. 后续竞品调研也沿用本规范

Claude Code、Codex、Trae、Reasonix、Hermes、CodeWhale 的调研也应该遵循同一标准：

```text
官方源代码 > 官方 Docs > 官方博客/模型卡 > 可信第三方 > 工程推论
```
