# 05. UI/UX 与原型说明

## 1. 当前状态

确定：

- UI/UX 当前只保留信息架构、导航模型、关键页面线框和交互原则。
- 不采用已被降级的视觉草稿。
- 高保真设计必须在线框与核心流程确认后重新生成或重新设计。

## 2. 产品体验原则

DeepSeekAgent 的 UI 不是聊天窗口包装，而是任务工作台。

体验目标：

- 每个页面只有一个主任务。
- 左侧导航随模式变化。
- 右侧面板服务当前上下文，而不是固定仪表盘。
- 高风险动作必须二次确认。
- 加载状态必须说明当前阶段，不允许只有“正在思考”。
- 错误必须给原因和恢复路径。
- 成本、cache、route、evidence 必须可见。

## 3. 桌面 Shell

标准布局：

```text
macOS titlebar
├─ Left Rail: Brand / Mode Switcher / Mode Context / Global Tools
├─ Topbar: workspace / search / help / notice / account
├─ Main Content: current primary task
└─ Context Panel: task-specific tabs
```

建议：

- 左侧宽度 260-300px。
- 右侧上下文宽度 320-380px。
- 中间主区承担唯一主任务。
- 顶栏不得重复放设置入口。

## 4. 模式导航

### 4.1 Mode Switcher

固定三项：

```text
Code
助理
连接手机
```

说明：

- `Code` 是 0.2.x 主线。
- `助理` 是 0.3.x 主线。
- `连接手机` 只能作为后续审批/通知/远程触发入口，不是完整移动端承诺。

### 4.2 Code 模式

左侧上下文：

```text
工作区
新建会话
新建需求
计划
Todo
变更
代码审查
成本与缓存
Code 插件
Code 定时任务
```

关键页面：

- Code 首页 / 工作台。
- 新建需求。
- 计划列表。
- 计划详情 / 执行。
- 文件变更审查。
- 成本与缓存。

验收：

- 当前工作区可见。
- 分支和状态可见。
- diff 和 approval 是核心路径，不隐藏到次级入口。
- Todo、计划、变更、审批有清晰归属。

### 4.3 助理模式

左侧上下文：

```text
新建任务
常用任务
历史任务
Todo
自动化任务
结果归档
成本与缓存
```

常用任务：

- 整理文件夹。
- 阅读资料。
- 写文档。
- 做调研。
- 分析表格。
- 创建自动化任务。

验收：

- 不显示代码仓库上下文，除非任务绑定了某个文件夹。
- artifact 必须带来源。
- 自动化入口不得绕过安全审批。

### 4.4 连接手机模式

左侧上下文：

```text
扫码连接
手机审批
消息提醒
远程触发
已连接设备
授权记录
安全设置
```

边界：

- 只做审批、提醒、远程触发和进度查看。
- 不做完整移动端。
- 不允许手机端构造新的高风险动作，只能处理预生成动作。

## 5. 关键线框摘要

### 5.1 Code 首页

主任务：

- 打开代码仓库。
- 新建需求。
- 继续计划。
- 查看最近任务、缓存命中、成本节省。

右侧：

- Todo。
- 变更。
- 审批。
- 成本。

### 5.2 新建需求

主表单：

- 背景。
- 目标。
- 验收标准。
- 约束。
- 参考资料。

辅助能力：

- 保存草稿。
- 让 Agent 补全需求。
- 生成实施计划。

### 5.3 计划详情

主区：

- 计划步骤。
- 当前步骤详情。
- 关联文件。
- 输出证据。
- 下一步建议。

右侧：

- 审批请求。
- 风险等级。
- checkpoint。
- allow / deny / allow once。

### 5.4 文件变更审查

主区：

- 文件列表。
- 并排 diff。
- 变更说明。

右侧：

- 风险等级。
- 回滚点。
- 测试建议。
- 拒绝 / 允许一次 / 应用并继续。

## 6. 下一步原型规则

- 先确认 Code 工作台关键流程，再做高保真。
- 先做桌面 Code Workbench，不同时展开所有模式。
- 高保真图必须服务 release gate 和核心用户路径。
- 不生成批量视觉图作为决策依据。
- 原型验收优先级高于视觉风格。
