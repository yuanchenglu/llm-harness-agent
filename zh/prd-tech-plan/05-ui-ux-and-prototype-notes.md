# 05. UI/UX 与原型说明

## 1. 当前状态

确定：

- UI/UX 当前主线改为中文 B 端桌面工作台。
- 旧的 Code-first / consumer assistant / 三模式模型只作为历史参考或高级入口，不再作为首屏真相。
- Kun/Quinn 的侧栏、中央工作区、底部输入框和轻视觉语言是本轮硬参考。

## 2. 产品体验原则

DeepSeekAgent 的 UI 不是聊天窗口包装，也不是运维诊断面板，而是给财务、行政、产品经理和律师使用的中文 B 端任务工作台。

体验目标：

- 每个页面只有一个主任务。
- 左侧导航随模式变化。
- 右侧面板服务当前上下文，默认隐藏为任务详情抽屉，而不是固定仪表盘。
- 高风险动作必须二次确认。
- 加载状态必须说明当前阶段，不允许只有“正在思考”。
- 错误必须给原因和恢复路径。
- 成本、cache、route、evidence 必须可在高级详情中查看，但不得压过首屏业务任务。

## 3. 桌面 Shell

标准布局：

```text
macOS titlebar
├─ Left Sidebar: 项目 / 任务列表 / 技能 / 自动化 / 设置
├─ Main Content: 当前任务 / 空态 / 模板 / 结果流
├─ Bottom Composer: 业务任务输入框
└─ Right Drawer: 任务详情 / 审批 / 回滚 / 高级诊断
```

建议：

- 左侧宽度 260-300px。
- 右侧抽屉宽度 360-440px，默认隐藏。
- 中间主区承担唯一主任务。
- 顶栏只放当前项目和必要状态，不放工程诊断堆栈。

## 4. 模式导航

### 4.1 默认导航

默认五项：

```text
项目
任务列表
技能
自动化
设置
```

说明：

- `项目` 和 `任务列表` 是首屏主路径。
- `技能` 只展示可用能力，不展示插件市场。
- `自动化` 保留本地预览边界。
- `设置` 承载版本、数据位置和诊断。
- `Code` 降级为高级入口。
- `连接手机` 本轮不展示为可用入口。

### 4.2 B 端任务模式

左侧上下文：

```text
项目
新建任务
任务列表
常用模板
技能
自动化
设置
```

关键页面：

- 首次打开 / 未选择项目。
- 已选择项目但无任务。
- 新建任务。
- 财务 / 行政 / 产品经理 / 律师模板。
- 任务运行中。
- 任务完成。
- 需要审批。
- 失败与恢复。
- 设置与诊断。

验收：

- 当前工作区可见。
- 用户可以从模板或底部输入框开始。
- diff、approval、rollback 在任务抽屉中清晰可见。
- route、cache、cost、raw JSON 默认隐藏到高级详情。

### 4.3 角色模板

常用模板：

- 财务：发票与报销核对、费用摘要、付款节点整理。
- 行政：会议纪要整理、制度文档整理、采购/资产台账摘要。
- 产品经理：需求拆解、PRD 评审、竞品资料整理。
- 律师：合同摘要、条款风险清单、材料整理。

验收：

- 未选择项目时不能运行任务。
- 模板只填充草稿，不自动提交。
- artifact 必须带来源。
- 未验证的 Excel/PDF/OCR/法律意见能力不能显示为可用功能。

### 4.4 高级 Code / 诊断

高级入口保留：

```text
Code / diff
审批
回滚
诊断
route / cache / cost / evidence
```

边界：

- 不作为默认首屏主路径。
- 不展示 API key、raw prompt、raw response 或 reasoning_content。
- 仍由 bridge 执行写入，UI 不直接写文件。

## 5. 关键线框摘要

### 5.1 B 端首页

主任务：

- 选择项目文件夹。
- 从模板或底部输入框新建任务。
- 查看最近任务。
- 查看需要审批的任务。

右侧：

- 默认隐藏。
- 选中任务后显示任务详情、审批、错误恢复和高级诊断。

### 5.2 新建任务

主表单：

- 目标。
- 背景。
- 验收标准。
- 约束。
- 参考资料。

辅助能力：

- 选择模板。
- 保存草稿。
- 运行只读任务。
- 进入需要审批的写入流程。

### 5.3 计划详情

主区：

- 计划步骤。
- 当前步骤详情。
- 关联文件。
- 输出证据。
- 下一步建议。

右侧：

- 审批请求。
- 风险等级。
- checkpoint。
- allow / deny / allow once。

### 5.4 文件变更审查

主区：

- 文件列表。
- 并排 diff。
- 变更说明。

右侧：

- 风险等级。
- 回滚点。
- 测试建议。
- 拒绝 / 允许一次 / 应用并继续。

## 6. 下一步原型规则

- 先确认 B 端首页、模板、底部输入框、任务抽屉和审批流程。
- 以用户给出的 Kun/Quinn 截图和 `/Volumes/Doc/Code/Kun` 源码作为硬参考。
- 高保真和实现都必须服务财务、行政、产品经理、律师的核心路径。
- 不生成新的不相关视觉方向。
- 原型验收优先级高于视觉风格。
